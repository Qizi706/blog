#### MIT 6.s081 基于 RISC-V 指令集架构的操作系统内核

时间：2026年2月 - 2026年4月 个人项目

_C, Linux_

**项目描述**：本项目以基于RISC-V指令集架构的微型操作系统xv6 为基础，对其内存管理，进程管理，文件系统，中断处理等模块进行扩展与优化。

**主要工作：**

1. 修改 xv6 的页表切换机制，简化用户空间和内核空间数据拷贝流程
2. 扩展 xv6 的物理内存分配机制，利用缺页中断实现物理页面的惰性分配和 COW 机制
3. 理解 xv6 的中断处理过程，基于中断机制实现 alarm 系统调用（注册定时器回调）
4. 理解 xv6 的进程调度以及上下文切换过程，以此为参照实现非抢占式调度的用户级线程
5. 更改数据结构和锁策略，在物理页分配与释放过程中使用更细粒度的锁以减少争用
6. 扩展 xv6 的文件系统，实现三级间接索引以支持更大的文件大小，实现软链接
7. 实现了 mmap 系统调用以支持内存映射文件，减少磁盘访问次数

#### CMU15-445 基于C++开发的支持简单 SQL 操作的单机数据库

时间：2025年11月 - 2025年12月 个人项目

_C++_

**项目描述：**这是一个面向磁盘的数据库管理系统（DAMS），名为 bustub。实现了缓冲池管理，B+ 树数据库索引，SQL 查询的操作执行器，多版本并发控制等。

**主要工作：**

1. LRU-K 页面替换机制，记录内存页的前 K 次访问时间戳来决定页面的淘汰顺序
2. 磁盘调度程序，将磁盘操作与主线程逻辑解耦，避免了程序在等待 I/O 时发生阻塞
3. 线程安全的缓冲池管理器，负责 4KB 数据帧的内存分配以及与磁盘的数据交换
4. B+ 树索引，支持线程安全的搜索、插入、删除（包括节点拆分和合并），以及用于支持按序叶节点扫描的迭代器
5. 实现执行器，支持 SELECT, DELETE, UPDATE, JOIN, AGGREGATION, LIMIT, DISTINCT 操作
6. 使用火山模型编写优化器规则，将包含索引列查询的顺序扫描自动转换为索引扫描，查询计划能在连接条件为等值且使用AND连接的场景下，自动将低效的嵌套循环连接转化为高效的哈希连接
7. 在数据库系统中实现了基于多版本并发控制（MVCC）的事务模块，通过维护数据的版本链和撤销日志，支持了快照隔离级别的多线程并发读写
